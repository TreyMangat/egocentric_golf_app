# First-run setup

Reference for getting this repo from `git clone` to a working local pipeline.
Terse on purpose — every command is meant to be copy-pasted. For the *why*
behind the architecture, see [`PROJECT_SPEC.md`](../PROJECT_SPEC.md);
for the capture-surface tradeoff, see [`capture-surface.md`](capture-surface.md).

## What this is

A personal egocentric golf-swing capture and analysis pipeline: phone records →
S3 → Temporal workflow segments by audio impacts → pose inference → metrics →
Mongo → Next.js dashboard. The architecture mirrors a production data-infra
stack at hobbyist scale (~$0/month with `LOCAL_DEV=1`). Full design and
phasing in [`PROJECT_SPEC.md`](../PROJECT_SPEC.md).

## Prerequisites

| Tool          | Version       | macOS                              | Windows                                       |
|---------------|---------------|------------------------------------|-----------------------------------------------|
| Python        | 3.11+         | `brew install python@3.11`         | `winget install Python.Python.3.11`           |
| Node.js       | 18+ (20 LTS)  | `brew install node@20`             | `winget install OpenJS.NodeJS.LTS`            |
| ffmpeg        | any recent    | `brew install ffmpeg`              | `winget install Gyan.FFmpeg`                  |
| git           | any           | `brew install git`                 | `winget install Git.Git`                      |
| Temporal CLI  | latest        | `brew install temporal`            | download from github.com/temporalio/cli/releases (no first-party winget package) |

Accounts:
- **MongoDB Atlas** — free tier (M0). https://cloud.mongodb.com/
- **AWS** — free tier covers everything in this repo. https://aws.amazon.com/

Skip Modal for now — `LOCAL_DEV=1` runs pose on your CPU.

## Clone and bootstrap

```bash
git clone https://github.com/treymangat/egocentric_golf_app.git
cd egocentric_golf_app/golf-pipeline

# Backend venv
cd backend
python3.11 -m venv .venv
source .venv/bin/activate              # macOS / Linux
# .venv\Scripts\Activate.ps1           # Windows PowerShell
pip install -e ".[dev]"
cd ..

# Web deps
cd web && npm install && cd ..
```

The mediapipe pin (`==0.10.14`) is intentional — newer releases dropped the
`solutions` API the pose code uses. Don't bump it without migrating to
`mediapipe.tasks.python.vision.PoseLandmarker`.

## Cloud setup

### MongoDB Atlas (M0, free)

1. Sign in at https://cloud.mongodb.com/.
2. **Create a Project** → name it anything.
3. **Build a Database** → **M0 Free**.
4. Provider **AWS**, Region **us-west-2 (Oregon)**, cluster name `Cluster0`. Create.
5. Atlas prompts for a database user:
   - Username: `golf-app`
   - **Autogenerate Secure Password** → copy it now (only chance).
   - Authentication method: Password.
6. Network Access prompt: **Add My Current IP Address**. Repeat this step every time you change networks.
7. Cluster up → **Connect → Drivers → Python → Copy connection string**:
   ```
   mongodb+srv://golf-app:<password>@cluster0.xxxx.mongodb.net/?retryWrites=true&w=majority
   ```
   Substitute `<password>` with the autogenerated one. Use this verbatim as
   `MONGO_URI` in `.env`.

For vector search (V1.5), see [`atlas-vector-index.md`](atlas-vector-index.md).

### AWS S3 + IAM user

1. Sign in at https://console.aws.amazon.com/.
2. **Region picker (top-right) → US West (Oregon) us-west-2**. Stay in this region for every step below.
3. **S3 → Create bucket**:
   - Name: `<your-name>-golf-pipeline` (globally unique)
   - Region: us-west-2
   - Block Public Access: **on** (default)
   - Versioning / tags / encryption: defaults
   - Create.
4. **IAM → Users → Create user**:
   - Name: `golf-pipeline-dev`
   - Permissions: **Attach policies directly → AmazonS3FullAccess**
   - Create.
5. Open the new user → **Security credentials → Create access key**:
   - Use case: **Local code**
   - Acknowledge → Create.
   - Copy the **Access key ID** and **Secret access key** now (secret is shown once).

> S3FullAccess is broad on purpose for V1. Tighten to `s3:PutObject` / `s3:GetObject` / `s3:DeleteObject` on the specific bucket prefix when you move beyond personal use.

## `.env`

```bash
cd golf-pipeline/backend
cp .env.example .env
```

Fill in `.env`:

```ini
# AWS (from the IAM + S3 steps above)
AWS_REGION=us-west-2
AWS_ACCESS_KEY_ID=AKIA...
AWS_SECRET_ACCESS_KEY=...
S3_BUCKET=your-name-golf-pipeline

# Mongo (from Atlas step 7, password substituted)
MONGO_URI=mongodb+srv://golf-app:<password>@cluster0.xxxx.mongodb.net/...
MONGO_DB=golf_pipeline

# Local-CPU pose, no Modal needed
LOCAL_DEV=1
```

Leave the Temporal block at its defaults (`localhost:7233`). Skip the Modal
lines — they're only read when `LOCAL_DEV=0`.

The repo-root `.gitignore` covers stray `.env` files; the canonical location
is `golf-pipeline/backend/.env`.

## Verify

```bash
cd golf-pipeline/backend
source .venv/bin/activate
python scripts/verify_setup.py
```

Expected: green `[ OK ]` on every required env var plus ffmpeg, AWS S3, Mongo,
Temporal. The Modal check prints `[SKIP]` because `LOCAL_DEV=1`. Exit code 0.

Anything red → [Common errors](#common-errors).

## Run locally

Four terminals. Activate the backend venv in each backend terminal.

**1. Temporal dev server** (port 7233, UI on 8233):
```bash
temporal server start-dev
```

**2. Temporal worker:**
```bash
cd golf-pipeline/backend && source .venv/bin/activate
python -m golf_pipeline.temporal.worker
```

**3. FastAPI** (port 8000):
```bash
cd golf-pipeline/backend && source .venv/bin/activate
uvicorn golf_pipeline.api.server:app --reload --port 8000
```

**4. Web** (port 3000):
```bash
cd golf-pipeline/web
npm run dev
```

Open http://localhost:3000. Sessions list will be empty until you run the smoke test.

## Smoke test

End-to-end run against synthesized fake-range audio. Confirms the full plumb
(presigned upload → S3 → workflow → audio segmenter → CPU pose → metrics →
Mongo) before you record a real swing.

```bash
cd golf-pipeline/backend && source .venv/bin/activate
python scripts/test_local_pipeline.py
```

Default: 15 s synthesized session with two planted impacts. Finishes in ~30–60 s
on a laptop CPU. Successful output:
- `workflow finished in Ns`
- a JSON `Mongo swings in session …` block with two swings.

The metrics will mostly be NaN — the "video" is a black frame, MediaPipe finds
no body. That's expected; this script tests **wiring**, not analytics.
Refresh http://localhost:3000 to see the new session appear.

## Phone capture (PWA)

Brief; full tradeoffs and rationale in [`capture-surface.md`](capture-surface.md).

1. Deploy the web app: `cd web && vercel deploy --prod`.
2. Expose FastAPI over HTTPS — Cloudflare Tunnel is the path of least resistance:
   ```bash
   cloudflared tunnel --url http://localhost:8000
   ```
3. In Vercel project settings, set `NEXT_PUBLIC_API_BASE` to the tunnel URL → redeploy.
4. On iPhone: open the Vercel URL in Safari → **Share → Add to Home Screen**.
5. Tap the home-screen icon → allow camera + mic on first run → record.

iOS Safari will not grant `getUserMedia` over plain HTTP outside `localhost`,
so HTTPS on both web and API is non-negotiable.

## Common errors

| Symptom | Cause | Fix |
|---|---|---|
| `verify_setup.py` Mongo: `ServerSelectionTimeoutError` | Your current IP isn't on Atlas's allowlist (changed networks since setup) | Atlas → Network Access → Add Current IP |
| `verify_setup.py` AWS S3: `EndpointConnectionError` or `403 Forbidden` | `AWS_REGION` in `.env` doesn't match the bucket's region | Set `AWS_REGION=us-west-2` (or wherever the bucket lives) |
| Worker / smoke test: `ffmpeg: command not found` | ffmpeg not on `PATH` | Reinstall via `brew` / `winget`, **restart the terminal** so `PATH` reloads |
| `OSError: [Errno 48] Address already in use` on **7233** | A previous `temporal server start-dev` is still running | `lsof -i :7233` (macOS/Linux) or `netstat -ano \| findstr 7233` (Windows); kill the PID |
| Same on **8000** | Stale uvicorn | Kill it, or `uvicorn ... --port 8001` and update `NEXT_PUBLIC_API_BASE` |
| Same on **3000** | Another Next.js dev server | `npm run dev -- --port 3001` |
| `ModuleNotFoundError: No module named 'mediapipe.solutions'` | A `pip install` ignored the pin and pulled mediapipe ≥ 0.10.18 (Solutions API removed) | `pip install -e ".[dev]" --force-reinstall mediapipe==0.10.14` |
| Workflow stuck in **Running** forever | Worker isn't connected, or task queue mismatch | Check terminal 2 is alive; verify `TEMPORAL_TASK_QUEUE` in `.env` matches what the worker registers |
| Upload activity: `NoSuchBucket` | `S3_BUCKET` typo or bucket in wrong region | Fix `.env`, restart worker + uvicorn |
| iOS Safari: camera prompt never appears | App served over HTTP (not HTTPS) outside `localhost` | Use Vercel + Cloudflare Tunnel, not raw IP |
